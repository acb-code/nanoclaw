## Breaking changes upstream has flagged

Extracted from upstream `CHANGELOG.md` diff:

1. **Pino logger replaced with built-in logger.** WhatsApp fork must be re-merged for the Baileys compatibility fix:
   ```
   git remote add whatsapp https://github.com/qwibitai/nanoclaw-whatsapp.git   # if missing
   git fetch whatsapp main
   git merge whatsapp/main
   ```
   This fork is not currently using WhatsApp (Telegram is the channel), so this is a **no-op** — skip unless WhatsApp is re-added.

2. **OneCLI Agent Vault replaces built-in credential proxy.**
   - Check runtime: `grep CONTAINER_RUNTIME_BIN src/container-runtime.ts`
   - If `'docker'` → run `/init-onecli` after merge to install OneCLI and migrate `.env` credentials to the vault.
   - If `'container'` (Apple Container) → re-merge `skill/apple-container` then run `/convert-to-apple-container`. Do NOT run `/init-onecli`.
   - This affects our custom `src/oas-proxy.ts` wiring: confirm the OAS proxy still receives credentials under the new vault model. OneCLI forwards `ONECLI_API_KEY` — ensure the OAS proxy's auth model doesn't collide.

## Recommended path: full merge, not cherry-pick

Reasoning:
- 453 commits is too many to triage individually.
- The interdependent refactors (logger swap, OneCLI vault, session cleanup, channel restructuring) don't cherry-pick cleanly.
- The fork already did a full merge at `40d6776`; that's the proven pattern.
- Rebase is worse: conflicts resolve per-commit, multiplying effort across 54 local commits.

## Execution plan

### Step 0 — Preflight
```bash
git status --porcelain                 # must be empty
git fetch upstream --prune
git remote -v                          # confirm upstream = github.com/qwibitai/nanoclaw.git
```

### Step 1 — Safety net
```bash
HASH=$(git rev-parse --short HEAD)
TS=$(date +%Y%m%d-%H%M%S)
git branch backup/pre-update-$HASH-$TS
git tag pre-update-$HASH-$TS
git push origin backup/pre-update-$HASH-$TS
git push origin pre-update-$HASH-$TS
```

### Step 2 — Merge
```bash
git merge upstream/main --no-edit
# Expect 26 conflicts (see list above).
```

### Step 3 — Resolve in this order

1. **Auto-resolvable "ours":** `git rm` the two workflow files; `git add` the telegram files.
2. **Auto-resolvable "theirs":** take upstream for `CHANGELOG.md`, `README.md`, `CONTRIBUTORS.md`, `docs/SECURITY.md`, `.claude/skills/setup/SKILL.md`, `repo-tokens/badge.svg` (or regenerate).
3. **Hand-merge in this order (dependency chain):**
   1. `package.json` + `package-lock.json` — `git checkout --theirs package-lock.json && npm install` to regenerate; hand-merge `package.json`.
   2. `.env.example` — union of both.
   3. `src/config.ts` — union of config surface.
   4. `src/db.ts` — upstream migrations first, then any fork additions.
   5. `src/container-runtime.ts` — adopt OneCLI runtime detection.
   6. `src/container-runner.ts` — this is the hardest file. Keep fork mounts (`~/sim-workdir`, cloud storage, OAS proxy), layer OneCLI vault integration on top.
   7. `src/container-runner.test.ts` — mirror above.
   8. `src/index.ts` — upstream orchestrator changes + OAS proxy registration + Hydra.
   9. `src/ipc.ts` — upstream auth/cleanup wins, fork additions layered.
   10. `setup/register.ts` — upstream wins unless fork added steps.
   11. `container/agent-runner/src/index.ts` + `container/agent-runner/src/ipc-mcp-stdio.ts` — take upstream structure, confirm fork behavior (if any) still holds.
   12. `CLAUDE.md`, `groups/main/CLAUDE.md` — keep Hydra persona, adopt new skill entries.

### Step 4 — Validate
```bash
npm install
npm run build
npm test
./container/build.sh    # container side effects: OneCLI, agent-runner changes
```

If `container/build.sh` steps use cached context, follow the note in root `CLAUDE.md` and prune the builder.

### Step 5 — Breaking-change migration
- Telegram fork: no action (no WhatsApp).
- Credentials: run `/init-onecli` (Docker) **or** re-merge `skill/apple-container` + `/convert-to-apple-container` (Apple Container). Check `src/container-runtime.ts` first.
- Re-test OAS proxy end-to-end after credential migration.

### Step 6 — Skill updates
`git branch -r --list 'upstream/skill/*'` → many exist. Optionally run `/update-skills` after the merge.

### Step 7 — Restart the service
```bash
# macOS
launchctl kickstart -k gui/$(id -u)/com.nanoclaw
# Linux
systemctl --user restart nanoclaw
```
